## Key Decisions

- **Soft-delete for Remove**, not hard-delete. `removeAccountFromCreator` flips `is_active=false`. Preserves audit trail + history; `getProfilesForCreator` filters them out so the UI hides them. Recoverable if someone reverses course.
- **`SiX` for Twitter** (X branding) per Simple Icons. Fanvue/Fanplace/Beacons stay on lucide because no Simple Icon exists.
- **Re-run / Retry consolidation** kept `RerunDiscoveryButton` with a `variant` prop and made `FailedRetryButton` a thin wrapper around it. The CreatorCard failed-state pill is divergent enough visually to keep its own inline render — only the label was unified.
- **Memory + skill, not memory alone**, for the autonomous-fix-list workflow. Memory file lives at `~/.claude/projects/.../memory/feedback_autonomous_fix_lists.md` as soft context; the skill file at `.claude/skills/autonomous-fix-list/SKILL.md` is the durable invocable artifact. Skill survives memory truncation, surfaces in `/skills` dialog, and is version-controlled with the project.
- **Schema NOT touched for `gothasiansclub`** (Simon's "group/brand" account question). The new Remove action gives him the manual lever to un-link the wrong attribution. Multi-creator profile linking is a real Phase 4+ design conversation, not a Phase 2 fix.
- **`tiktok_shop` icon uses lucide `ShoppingCart`** (subagent's deviation from spec which said "reuse SiTiktok"). Visually differentiates the Shop row from the social TikTok row. One-line swap if Simon wants parity.
- **No new dependencies beyond `react-icons`** — kept the surface tight.
- **Combined spec + code-quality reviews** for tasks whose plan was verbatim code; split for judgment-heavy tasks. Saved several subagent dispatches across the run.
