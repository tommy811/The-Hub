## What We Did

### Universal URL Harvester (the whole session)

**Architecture:** new `scripts/harvester/` package, single entry `harvest_urls(url, supabase) -> list[HarvestedUrl]`. 3-tier cascade:

1. **Cache** (`harvester/cache.py`) — new `url_harvest_cache` table (workspace-agnostic, 24h TTL, mirrors the `classifier_llm_guesses` no-RLS pattern). On cache hit, returns immediately and skips Tiers 1+2.
2. **Tier 1** (`harvester/tier1_static.py`) — httpx + BeautifulSoup. Pulls the page, extracts anchors, runs a 4-signal escalation detector: SPA marker (astro-island, next-data, single-div pattern), near-empty anchor count (`< 3` resolved hrefs), sensitive-content gate keywords ("over 18", "i agree", "open link", etc.), JS-only body (no rendered text). Any signal trips → escalate.
3. **Tier 2** (`harvester/tier2_headless.py` + `harvester/page_function.js`) — `apify/puppeteer-scraper` actor. The page function:
   - Hooks `window.open` and `location.href` setters BEFORE page scripts execute (so SPA single-page redirectors get captured even if their click handler tries to navigate the entire window).
   - Auto-clicks 7 interstitial keyword variants ("open link", "continue", "i am over 18", "i agree", "i confirm", "18+", "enter") via Puppeteer 22+ `page.$$('xpath/...')` selector syntax.
   - Returns the unified anchor + intercepted-navigation set.

After harvest, each URL is canonicalized + classified via the existing `pipeline.classifier`, then mapped through a host-aware `_destination_class_for(account_type, canonical_url)` that promotes Substack subdomains / Spotify / Apple Podcasts → `content`; amzn.to / geni.us / lnk.to / shareasale → `affiliate`; Shopify subdomains / Etsy / Depop → `commerce`; Telegram / WhatsApp / Discord → `messaging`. Same-host self-links are dropped.

**Schema changes (4 migrations applied via Supabase MCP):**
- `20260426000000_url_harvester_v1` — adds `url_harvest_cache` table (23 → 24 tables); adds 3 audit columns to `profile_destination_links` (`harvest_method`, `raw_text`, `harvested_at`).
- `20260426010000_commit_discovery_result_v3_harvester_audit` — RPC writes the new audit fields.
- `20260426020000_extend_destination_class_check` — extends the CHECK constraint from 4 → 10 values (`monetization|aggregator|social|commerce|messaging|content|affiliate|professional|other|unknown`). TEXT-with-CHECK pattern, not Postgres ENUM.
- `20260426030000_commit_discovery_result_v4_update_destination_class` — patches the ON CONFLICT clause to update `destination_class` so post-fix re-runs refresh stale rows.

**Resolver wiring** — `pipeline/resolver.py::_classify_and_enrich` now delegates to `harvest_urls()` (one call per page returns all outbound destinations) instead of dispatching per-aggregator. Deleted `scripts/aggregators/` entirely.

**UI** — Creator HQ renders new "All Destinations" section grouped by `destination_class` with a `gated` chip on rows where `harvest_method='headless'` (so you can spot at a glance which destinations required the Tier 2 escalation).

**Gazetteer** — ~30 new rules in `data/monetization_overlay.yaml` covering BuyMeCoffee, Ko-fi, Substack subdomains, Spotify, Telegram, WhatsApp, Apple Podcasts, affiliate redirectors.

**Canonicalizer extension** — strips additional tracking params (`igsh`, `l_`, `s`, `_t`, `aff`, `ref_id`) so `?s=21` and `?igsh=...` no longer create false-distinct rows in `profile_destination_links`.

- **Messaging class mapping** — Telegram URLs were initially landing as `other`; added the messaging mapping to the host-aware promoter (same commit).

## Key Decisions

- **Single universal entry, not per-aggregator extractors.** The old `aggregators/{linktree,beacons,custom_domain}.py` model didn't generalize to novel aggregator platforms (link.me, allmylinks, tapforallmylinks, custom Astro pages). One `harvest_urls()` call handles every page and the 3-tier cascade decides cost. Deleted the entire `aggregators/` package.
- **TEXT + CHECK, not Postgres ENUM, for `destination_class`.** Forward-compat additions are a single `DROP CONSTRAINT / ADD CONSTRAINT` swap instead of `ALTER TYPE ... ADD VALUE` ceremony. Worth it given we extended 4 → 10 in this session and likely will again.
- **Page function hooks `window.open` + `location.href` BEFORE page scripts run.** Many sensitive-content gates do `location.href = '...'` instead of opening an anchor, so a passive crawler misses them entirely. Hooking BEFORE scripts execute is the only reliable capture.
- **Auto-click 7 interstitial keyword variants.** Empirically derived from the live smoke (the OnlyFans-adjacent corner of the web is full of these gates). `page.$$('xpath/...')` selector syntax works in Puppeteer 22+.
- **Cache TTL = 24h, workspace-agnostic.** Mirrors `classifier_llm_guesses`. Aggregator pages don't change minute-to-minute and the dedup wins (re-running discovery on a known creator is now ~free for the harvester layer).
- **`gated` chip in UI**, not a hard error. A `harvest_method='headless'` row is a signal that the destination required active interception — useful UX for "is this a real CTA the creator owns, or did we follow a hostile gate". Not blocking.
- **Did not modify `scripts/.env`.** `SUPABASE_DB_URL` is still unfilled and SCHEMA.md regen still blocks on it. PROJECT_STATE.md §4.1 hand-updated to 24 tables; cosmetic discrepancy with `docs/SCHEMA.md`. Simon's call.
